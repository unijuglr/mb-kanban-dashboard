### Why the proof harness is failing
The current Neo4j client now uses batched `UNWIND $entities AS entity` writes in `batch_merge(...)`, while `scripts/prove-mb-079.py` still looks for the older per-entity `MERGE (e:Entity {olid: $olid})` request shape and expects:
- `merged == 2`
- `captured_request_count == 6`
- `merge_requests_seen == 2`

That expectation is stale against the current client behavior, so the proof script no longer serves as a trustworthy executable proof.
